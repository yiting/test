## 解决什么问题
1. 程序依照设计稿生成ui界面代码（支持多种语言）。
2. 减少视觉设计师与开发联合走查设计还原的工作量。
3. 生成页面直接供前台使用，减少ui还原的环节。

## 目录结构
目前SparkX项目集成了一些我们微信数据平台业务开发过程中的一些类库，这个类库也适用于公司其他BG的业务，代码组织结构如下，就当抛砖引玉，欢迎后续各BG业务童鞋分享。
```
├── main
│   ├── java
│   │   └── com
│   │       └── tencent
│   │           └── sparkx
│   │               └── tool
│   ├── proto
│   │   └── sample.proto
│   ├── resources
│   └── scala
│       └── com
│           └── tencent
│               └── sparkx
│                   ├── access          # 包含各存储系统的访问类库
│                   ├── app             # Spark应用程序目录
│                   ├── common          # 一些公用的类库
│                   ├── Logging.scala   # 日志基类
│                   └── util            # 一些工具类方法
└── test
    └── scala
        └── com
            └── tencent
                └── sparkx
                    ├── SparkXFunSuite.scala    # 单元测试基类
                    └── util                     
```
